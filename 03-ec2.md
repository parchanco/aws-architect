# 03 - EC2 (Elastic Compute Cloud)

## Resumen Ejecutivo
Servidores virtuales en la nube. Núcleo de AWS compute.

## Tipos de Instancias

### Nomenclatura
```
t3.large
│ │  │
│ │  └─ Tamaño (micro, small, medium, large, xlarge...)
│ └──── Generación (1, 2, 3, 4...)
└────── Familia (tipo de uso)
```

### Familias Principales

| Familia | Uso | Ejemplo Odoo |
|---------|-----|--------------|
| **t** | General purpose burstable | Dev/test: t3.medium |
| **m** | General purpose balanced | Prod small: m5.large |
| **c** | Compute optimized | Batch processing |
| **r** | Memory optimized | Prod large: r5.xlarge |
| **i** | Storage optimized | NoSQL databases |

## Modelos de Compra

| Modelo | Descuento | Compromiso | Uso |
|--------|-----------|------------|-----|
| **On-Demand** | 0% | Ninguno | Dev, testing |
| **Reserved** | 40-75% | 1-3 años | Prod 24/7 |
| **Savings Plans** | 30-70% | $/hora 1-3 años | Prod flexible |
| **Spot** | 70-90% | Puede interrumpir | Batch jobs |

### Ejemplo Costos (t3.large eu-west-1)
```
On-Demand:  $0.0832/h × 730h = $60.74/mes
Reserved 1yr: $0.049/h × 730h = $35.77/mes (41% ahorro)
Spot: ~$0.025/h × 730h = $18.25/mes (70% ahorro)
```

## Security Groups
Firewall virtual para instancias.

```python
# Security Group típico Odoo
Inbound:
- Port 22 (SSH): Tu IP solamente
- Port 8069: Desde ALB Security Group
- Port 8072 (Longpolling): Desde ALB SG

Outbound:
- All traffic: 0.0.0.0/0 (o restringir)
```

**Reglas de oro**:
- Stateful (respuesta automática)
- Deny by default
- Referencia otros SGs cuando sea posible

## IPs

### IP Privada
- Interna a VPC (10.0.x.x, 172.16.x.x, 192.168.x.x)
- No cambia nunca
- Gratis

### IP Pública
- Accesible desde Internet
- **CAMBIA** al reiniciar
- Gratis

### IP Elástica
- IP pública **fija**
- No cambia aunque reinicies
- Gratis si está en uso
- ~$3.60/mes si está reservada sin usar

**Best Practice**: Usa Load Balancer en vez de Elastic IP.

## Placement Groups

### Cluster
- Todas instancias en mismo rack
- Ultra baja latencia
- Si rack falla → todo cae

### Spread
- Instancias en diferentes racks
- Alta disponibilidad
- Máximo 7 instancias/AZ

### Partition
- Para sistemas distribuidos masivos (Hadoop, Kafka)

**Para Odoo**: Normalmente no usar, o Spread si HA crítica.

## ENI (Elastic Network Interface)
Tarjeta de red virtual que puedes mover entre instancias.

**Uso principal**: Failover manual con IP fija.

## EC2 Hibernate
Guarda RAM a disco. Útil para dev environments que paras por la noche.
